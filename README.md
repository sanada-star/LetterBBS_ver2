# LetterBBS ver2

LetterBBS は、KENT WEB様で公開されている「Patio」をベースに、PBC（Play by Chat）などの「キャラクター交流」向けに大幅な改造を施したCGIスクリプトです。

往復書簡形式のロールプレイ、1対1のロールプレイ上の秘匿会話、そしてログの保存に特化しています。

## 主な機能

- **往復書簡特化機能**: 1対1（多対多も可）のキャラクター間での手紙のやり取りを想定した表示。
- **タイムライン（文通デスク）**: 特定の相手とのスレッド内のやり取りを抽出して、チャット風のタイムライン表示で閲覧可能。
- **SQLite3によるデータ管理**: 過去のテキストファイル管理からデータベース管理に移行し、大規模な利用でも安定して動作しやすくなっています（WALモード対応）。
- **管理画面 (`admin.cgi`)**: プログラムを開かなくても、ブラウザ上から表示設定やテーマ変更を一括で制御可能。
- **テーマ機能**: `admin.cgi` から簡単にCSSテーマを切り替え可能。

## 設置方法 (簡易)

アップロード後のディレクトリツリー構成と、新たに付与すべき**パーミッション（属性）**は以下の通りです。
`[ ]` 内の数字が推奨されるパーミッションです。

```text
patio/
 ├── admin.cgi     [705] ※実行権限
 ├── api.cgi       [705] ※実行権限
 ├── captcha.cgi   [705] ※実行権限
 ├── check.cgi     [705] ※実行権限
 ├── migrate.cgi   [705] ※実行権限
 ├── patio.cgi     [705] ※実行権限
 ├── init.cgi      [604] ※設定ファイル (init.cgi.sampleをリネーム)
 ├── data/         [707] ※データ保存ディレクトリ。書き込み権限が必要
 │    └── sessions/ [707] ※ここも必要
 ├── upl/          [707] ※画像アップロードディレクトリ。書き込み権限が必要
 ├── cmn/          （アップロードしたままのデフォルトで問題ありません）
 ├── lib/          （アップロードしたままのデフォルトで問題ありません）
 └── tmpl/         （アップロードしたままのデフォルトで問題ありません）
```

> ※ `705` と `707` は さくらのレンタルサーバ での推奨設定です。サーバー環境によっては、`705` を `755` に、 `707` を `777` に読み替えて設定してください。
> ※ `init.cgi` のパーミッションは `604` または `644` となります。

## プログラムの設定及び設置 (詳細)

### STEP 1: 初期設定ファイルの準備 (`init.cgi`)

1. ダウンロードした `patio/` ディレクトリの中にある **`init.cgi.sample`** というファイルをコピーし、名前を **`init.cgi`** に変更します。
2. テキストエディタ（サクラエディタ、VSCodeなど）で `init.cgi` を開き、以下の設定を自身のサーバー環境に合わせます。

**📝 init.cgi で設定すること**

- **【重要】セキュリティ設定 (`$cf{csrf_secret}`)**
  - 33行目付近の `'CHANGE_ME_TO_RANDOM_STRING'` となっている箇所を、**必ず適当な英数字の羅列（ランダムな文字列）に変更** してください。この設定を怠るとセキュリティ上の脆弱性になります。
- **URL・パス設定**
  - `$cf{cgi_url}` や `$cf{admin_url}` を設定します。基本的にはデフォルトの相対パス（`./patio.cgi` 等）で問題なく動きますが、うまく画面遷移しない場合は `http://.../patio.cgi` のように絶対URLを入力してください。

> ⚠️ **注意: ほとんどの設定は `admin.cgi` から行います**
> 掲示板のタイトル、テーマ変更、1ページあたりの記事表示件数など、多くの設定項目は設置後に **`admin.cgi` (管理画面)** から変更します。
> `init.cgi` 内にも設定変数が記載されていますが、データベースの準備ができた後は管理画面の設定が優先される仕組みになっています。

### STEP 2: ファイルのアップロードとパーミッション設定

1. FTPソフトを使って、上記の「設置方法 (簡易)」の通りにファイルをサーバーにアップロードします。
2. `.cgi` ファイルや `data/`, `upl/` ディレクトリに、適切なパーミッション（属性）を設定します。

### STEP 3: データベースの生成と初期化

1. ブラウザから、設置した **`patio.cgi`** にアクセスします。
2. この初回アクセスのタイミングで、自動的にデータベースファイル（`data/letterbbs.db`）が生成され、必要な初期データが登録されます。

### STEP 4: 【必須】管理者パスワードの変更

インストールが終わったら、**一番最初に必ず** 以下の作業を行ってください。

1. ブラウザから **`admin.cgi`** にアクセスします。
2. 以下の初期情報でログインします。
   - **ログインID:** `admin`
   - **初期パスワード:** `password`
3. ログインできたら、管理画面の設定メニューから、**必ずパスワードを推測されにくいものに変更** してください。

### STEP 5: 基本設定（管理画面より）

パスワードを変更したら、引きつづき管理画面から掲示板のタイトルやテーマ等、必要な環境設定を行ってください。

## トラブルシューティング

**Q. `500 Internal Server Error` が出て画面が真っ白になる**

- `.cgi` ファイルのパーミッションが間違っている可能性があります（`705` または `755` になっているか確認）。
- `init.cgi` や各 CGI プログラムの1行目（`#!/usr/local/bin/perl`）が、ご利用のサーバーの Perl パスと合っていない可能性があります。さくらの場合はそのままでOKですが、環境によって `#!/usr/bin/perl` などに変更してください。
- ファイルをアップロードする際、改行コードが環境と合っていない（ASCIIモードでアップロードされていない）可能性があります。

**Q. 投稿やアップロードができない / SQLiteのエラーが出る**

- `data/` フォルダ、または `data/sessions/` フォルダのパーミッションが `707`（または `777`）になっていない可能性があります。
- サーバーの保存容量がいっぱいになっていないか確認してください。

**Q. デザインが崩れる / 画面表示がおかしい**

- `cmn/` フォルダ（CSS等）や `tmpl/` フォルダ（HTMLテンプレート）が正しくアップロードできているか確認してください。
- ブラウザのキャッシュが残っている可能性があります（強制リロードをお試しください）。

## デザインのカスタマイズ

本プログラムでは、管理画面（`admin.cgi`）の「一般設定」からデザインテーマを簡単に切り替えることができます。

用意されている主要なスタイルは以下の通りです。

- **Default**: 白背景の定番スタイル
- **cool**: 黒と青を基調としたクールなスタイル
- **Dark**: 黒背景に濃い赤のダークスタイル
- **punk**: サイバーパンクカラーなネオンスタイル
- **fox**: 狐をモチーフにした和テーマ

CSSなどを直接編集してオリジナルのスタイルを作りたい場合は、`cmn/css/` 内の各種設定や、`tmpl/` ディレクトリ配下のHTMLファイルを調整してください。

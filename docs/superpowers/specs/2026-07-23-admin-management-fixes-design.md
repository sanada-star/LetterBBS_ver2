# 管理画面残存不具合修正 設計書

## 目的

受入試験で再現した ADM-13、ADM-15、ADM-16 を、既存の管理画面構成とデータモデルを保ったまま修正する。

## 対象

- ADM-13: スレッド管理で現行スレッドと過去ログを切り替え、過去ログを復元できるようにする。
- ADM-15: 会員一覧に権限、状態、登録日を正しく表示する。
- ADM-16: 複数選択したスレッドに対し、削除、管理者コメント機能切替、過去ログ化、復元を全件適用する。

## 対象外

- 管理画面全体のデザイン変更
- データベーススキーマ変更
- 会員の権限・状態を編集する新機能
- 過去ログの仕様変更

## 設計

### ADM-13

`Admin::thread_list` が既に受け取っている `status=active|archived` を画面から選べるよう、スレッド管理画面に「現行スレッド」「過去ログ」の切替リンクを追加する。

- 現行スレッド表示時は、既存の「削除」「管理者コメント機能切替」「過去ログ化」を表示する。
- 過去ログ表示時は、「削除」「復元」を表示する。
- ページングや操作後のリダイレクトでは、現在の `status` を維持する。
- 不正な `status` は既存どおり `active` に正規化する。

### ADM-15

`Admin::member_list` で、モデルから取得した会員データに表示専用値を追加する。

- `rank_label`: rank 1を「閲覧のみ」、rank 2を「書込可」とする。未知値は元の数値を表示する。
- `status_label`: モデルが返す数値の `is_active` を使用し、1を「有効」、0を「無効」とする。未定義・未知値は空欄にせず元の値を表示する。
- `display_date`: `created_at` を管理画面既存形式 `YYYY/MM/DD HH:MM` に変換する。

テンプレートの既存変数名は変更しない。

### ADM-16

実運用の `admin.cgi` では同名パラメータが後の値で上書きされ、Controllerへ到達する前に複数IDが失われている。このため、URLエンコードとmultipartの両パーサーで同名値を配列として保持し、`AdminCGI::param` がリストコンテキストでは全値、スカラーコンテキストでは従来互換の単一値を返すようにする。

Controllerでは複数値取得を小さな補助処理へまとめる。`ids` と `post_ids` はリストコンテキストで全値を取得し、正の整数だけへ正規化してから処理する。

- 単一操作の `thread_id` は従来どおり使用できる。
- 複数選択時は選択順にかかわらず全IDへ適用する。
- 空値、不正値、重複値は無視する。
- 記事個別削除のスレッド所属確認、親記事削除禁止、削除済み記事除外は維持する。
- 通常の単一値パラメータについては、既存Controllerのスカラー取得動作を変更しない。

## エラー処理と安全性

- 既存のCSRF検証を維持する。
- 不正IDは処理せず、SQLへ渡さない。
- 操作後は対象一覧へ戻し、過去ログ操作では `status=archived` を維持する。
- 既存の削除確認ダイアログを維持する。

## テスト

テストを先に追加し、失敗を確認してから実装する。

- `controller_admin.t`
  - active／archivedの切替値と操作ボタンを確認する。
  - archived一覧の復元操作が `restore` として送信されることを確認する。
  - 会員の表示用3項目を確認する。
  - 複数の `ids` が削除・管理者コメント機能切替・過去ログ化・復元へ全件渡ることを確認する。
  - 単一 `thread_id` と不正・重複IDの扱いを確認する。
- `admin_cgi_params.t`
  - 実際のフォームエンコード文字列 `ids=1&ids=2` がリストコンテキストで2値となることを確認する。
  - multipartでも同名フィールドを複数保持できることを確認する。
  - スカラーコンテキストの既存単一値取得が維持されることを確認する。
- 全Perl、Node、Python回帰試験を再実行する。
- アップロード後、本番同等環境でADM-13、ADM-15、ADM-16を専用データにより再試験する。
